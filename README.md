# Anomaly Detection in Time Series using Foundation Models

## Описание

В данной работе исследуется возможность применения фундаментальных моделей временных рядов для задачи обнаружения аномалий в zero-shot режиме, то есть без дополнительного обучения на целевых данных.

Основная идея заключается в использовании ошибки прогноза в качестве меры аномальности.

---

## Идея метода

Пусть задан временной ряд:
\[
X = \{x_1, x_2, ..., x_T\}
\]

Метод работает следующим образом:

1. Ряд разбивается на перекрывающиеся окна длины context_len
2. Для каждого окна модель предсказывает следующее значение
3. Вычисляется ошибка прогноза:
\[
s(t) = ||x_{t+1} - \hat{x}_{t+1}||^2
\]
4. Ошибка используется как anomaly score
5. Итоговая оценка получается усреднением по перекрывающимся окнам

---

## Используемые модели

В работе используются предобученные foundation-модели временных рядов:

Chronos: токенизирует временной ряд и применяет трансформер аналогично языковым моделям  

Moirai: обрабатывает многомерные временные ряды с учётом межканальных зависимостей  

TimesFM: разбивает ряд на патчи и использует трансформер для прогнозирования  

TTM (Tiny Time Mixer): использует патчевое представление и MLP-mixer вместо attention  

Все модели используются в zero-shot режиме.

---

## Датасеты

Используются датасеты из библиотеки TimeEval:

NASA-SMAP, NASA-MSL - телеметрия космических аппаратов  
SMD - метрики серверов  
SWaT, WADI - данные индустриальных систем  

Каждый датасет представляет собой многомерный временной ряд вида:

| timestamp | feature_1 | feature_2 | ... | is_anomaly |
|----------|----------|----------|-----|------------|

feature_i - показания датчиков  
is_anomaly - разметка (используется только для оценки)

---

## Метрики качества

Используются стандартные метрики:

- Precision  
- Recall  
- F1-score  
- ROC-AUC  

Рассматриваются два режима оценки:

без point-adjustment (поточечная оценка)  
с point-adjustment (по интервалам аномалий)

---

## Основные параметры

- context_len - длина окна истории  
- prediction_len = 1 - одношаговый прогноз  
- batch_size - количество окон, обрабатываемых одновременно  

Параметры выбирались с учётом ограничений и обучающего режима моделей.

## Структура проекта

Основные файлы:

- datasets.py - отвечает за загрузку и подготовку данных  
- detector.py - реализует алгоритм детектирования аномалий  
- evaluation.py - считает метрики  
- model_factory.py - создаёт и настраивает модели  
- run_model.py - основной скрипт для запуска экспериментов  
- models/ - содержит обертки для используемых моделей
